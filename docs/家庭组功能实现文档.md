# 家庭组功能 - 实现逻辑与接口文档

## 一、功能概述

家庭组是 AI 记账应用的协作功能模块，允许用户创建/加入家庭组，实现多人共享账本。核心设计原则：

- **极简设计**：加入后所有账单自动归属家庭组，无需手动标记
- **弹窗交互**：家庭组管理通过弹窗完成，不设独立页面
- **角色区分**：组长（owner）可管理成员、解散家庭组；成员（member）可记账、查看汇总、退出

## 二、数据库设计

### 2.1 ER 关系图

```
User 1 ──── N FamilyMember N ──── 1 Family
  │                                  │
  └──── createdFamilies ─────────────┘
```

### 2.2 表结构

#### `families` 家庭组表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 主键 |
| name | String | NOT NULL | 家庭组名称，最长20字 |
| creator_id | String | FK → users.id | 创建者（组长） |
| invite_code | String | UNIQUE | 6位邀请码，大写字母+数字 |
| created_at | DateTime | DEFAULT now() | 创建时间 |
| updated_at | DateTime | UPDATED | 更新时间 |

#### `family_members` 家庭组成员表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | String (cuid) | PK | 主键 |
| family_id | String | FK → families.id | 所属家庭组 |
| user_id | String | FK → users.id | 成员用户 |
| role | String | DEFAULT 'member' | 角色：owner / member |
| joined_at | DateTime | DEFAULT now() | 加入时间 |

**唯一约束**：`(family_id, user_id)` — 一个用户在一个家庭组中只能有一条记录

### 2.3 Prisma Schema

```prisma
model Family {
  id         String   @id @default(cuid())
  name       String
  creatorId  String   @map("creator_id")
  inviteCode String   @unique @map("invite_code")
  createdAt  DateTime @default(now()) @map("created_at")
  updatedAt  DateTime @updatedAt @map("updated_at")

  creator  User           @relation(fields: [creatorId], references: [id])
  members  FamilyMember[]

  @@map("families")
}

model FamilyMember {
  id       String   @id @default(cuid())
  familyId String   @map("family_id")
  userId   String   @map("user_id")
  role     String   @default("member")
  joinedAt DateTime @default(now()) @map("joined_at")

  family Family @relation(fields: [familyId], references: [id])
  user   User   @relation(fields: [userId], references: [id])

  @@unique([familyId, userId])
  @@map("family_members")
}
```

### 2.4 业务约束

- 一个用户只能加入一个家庭组（通过 `family_members` 表的 userId 唯一性保证）
- 邀请码全局唯一，6位大写字母+数字（排除易混淆字符 I/O/0/1）
- 组长退出 = 解散家庭组（级联删除所有成员记录 + 家庭组记录）

## 三、API 接口

所有接口需携带 JWT Token（Bearer Authorization）。

### 3.1 创建家庭组

```
POST /family/create
```

**请求体：**
```json
{ "name": "我的家" }
```

**逻辑：**
1. 检查当前用户是否已加入家庭组 → 已加入则返回 409 CONFLICT
2. 生成6位邀请码（确保唯一）
3. 创建 Family 记录 + 同时创建 FamilyMember（role=owner）
4. 返回家庭组信息（含成员列表）

**响应：**
```json
{
  "code": 0,
  "message": "创建家庭组成功",
  "data": {
    "id": "clxxx",
    "name": "我的家",
    "creatorId": "user-id",
    "inviteCode": "A3B5K9",
    "createdAt": "2026-05-26T...",
    "updatedAt": "2026-05-26T...",
    "members": [
      {
        "id": "mem-id",
        "userId": "user-id",
        "role": "owner",
        "joinedAt": "2026-05-26T...",
        "user": { "id": "user-id", "nickname": "小明", "avatar": null, "phone": "138..." }
      }
    ]
  }
}
```

### 3.2 加入家庭组

```
POST /family/join
```

**请求体：**
```json
{ "inviteCode": "A3B5K9" }
```

**逻辑：**
1. 检查当前用户是否已加入家庭组 → 已加入则返回 409 CONFLICT
2. 根据 inviteCode 查找 Family → 不存在则返回 404 NOT_FOUND
3. 创建 FamilyMember（role=member）
4. 返回家庭组信息

**错误码：**
| 场景 | HTTP Status | 错误信息 |
|------|-------------|----------|
| 已加入家庭组 | 409 | 你已加入家庭组，请先退出后再加入新的家庭组 |
| 邀请码无效 | 404 | 邀请码无效，请检查后重试 |

### 3.3 获取我的家庭组

```
GET /family
```

**逻辑：**
1. 查询当前用户的 FamilyMember 记录
2. 不存在则返回 `null`
3. 存在则返回家庭组信息 + 成员列表 + myRole

**响应（已加入）：**
```json
{
  "code": 0,
  "message": "获取成功",
  "data": {
    "id": "clxxx",
    "name": "我的家",
    "creatorId": "user-id",
    "inviteCode": "A3B5K9",
    "members": [...],
    "myRole": "owner"
  }
}
```

**响应（未加入）：**
```json
{
  "code": 0,
  "message": "获取成功",
  "data": null
}
```

### 3.4 退出家庭组

```
POST /family/leave
```

**逻辑：**
1. 查找当前用户的 FamilyMember 记录 → 不存在则返回 404
2. 如果角色是 owner → 解散家庭组：
   - 删除所有 FamilyMember 记录
   - 删除 Family 记录
   - 返回 `{ dissolved: true }`
3. 如果角色是 member → 仅删除自己的记录：
   - 删除该 FamilyMember
   - 返回 `{ dissolved: false }`

### 3.5 移除成员

```
DELETE /family/member/:memberId
```

**逻辑：**
1. 查找当前用户的 FamilyMember → 不存在或非 owner 则返回 403 FORBIDDEN
2. 查找目标 memberId 的 FamilyMember → 不存在或不属于同一家庭组则返回 404
3. 不能移除 owner 角色 → 返回 400 BAD_REQUEST
4. 删除目标 FamilyMember 记录

## 五、邀请码生成策略

```typescript
function generateInviteCode(): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'; // 排除 I/O/0/1
  let code = '';
  for (let i = 0; i < 6; i++) {
    code += chars[Math.floor(Math.random() * chars.length)];
  }
  return code;
}
```

- 6位长度，32个可选字符，共 32^6 = 1,073,741,824 种组合
- 排除易混淆字符（I/O/0/1），避免口头传达时出错
- 创建时检查唯一性，冲突则重新生成
