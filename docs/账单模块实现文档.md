# 账单模块实现文档

## 一、功能概述

账单模块是系统的核心功能，采用**对话式记账**为核心交互方式，同时保留手动记账入口。

### 核心流程

**对话记账流程（确认后入库）：**

```
用户在首页输入消费描述
    ↓
本地关键词解析（提取金额、分类、类型等）
    ↓
解析成功 → AI回复确认卡片（解析结果存metadata，不创建账单）
    ↓
用户点击"确认记账" → 创建账单入库 → AI回复"已记录 ✓"
```

```
解析失败 → AI回复引导消息（提示用户如何输入或使用手动记账）
```

**手动记账流程：**

```
用户点击"手动记账"按钮 → 弹出记账弹窗 → 选择分类/填写金额 → 提交
```

### 设计要点

- 首页就是AI对话界面，所有记账都通过对话完成
- AI解析后展示**结构化确认卡片**，用户一键确认或取消
- 确认卡片是对话中的结构化消息，不是弹窗
- 解析结果带**置信度**（high/medium/low），低置信度时提示用户确认
- 手动记账按钮保留在标题栏，作为备选入口
- 今日收支卡片浮动在对话区顶部，记账后实时更新

## 二、数据库设计

### 2.1 Bill 表

| 字段        | 类型          | 说明                                          |
| ----------- | ------------- | --------------------------------------------- |
| id          | String (cuid) | 主键                                          |
| user_id     | String        | 创建者 ID，关联 users 表                      |
| family_id   | String?       | 家庭组 ID，关联 families 表，可为空           |
| category_id | String        | 分类 ID，关联 categories 表                   |
| type        | String        | 账单类型：`expense`（支出）/ `income`（收入） |
| amount      | Decimal(12,2) | 金额，最大支持 9999999999.99                  |
| note        | String?       | 备注                                          |
| account     | String?       | 账户标签，如"微信"、"支付宝"                  |
| date        | DateTime      | 记账日期                                      |
| source      | String        | 来源：`manual`（手动）/ `ai`（AI对话）        |
| created_at  | DateTime      | 创建时间                                      |
| updated_at  | DateTime      | 更新时间                                      |

### 2.2 Category 表

| 字段      | 类型          | 说明                               |
| --------- | ------------- | ---------------------------------- |
| id        | String (cuid) | 主键                               |
| user_id   | String?       | 用户 ID，null 表示系统默认分类     |
| name      | String        | 分类名称                           |
| type      | String        | 分类类型：`expense` / `income`     |
| icon      | String        | 图标 emoji                         |
| is_system | Boolean       | 是否系统默认分类，系统分类不可删除 |

**唯一约束：** `(user_id, name, type)` — 同一用户下同类型分类名称唯一

### 2.3 ChatMessage 表

| 字段       | 类型          | 说明                             |
| ---------- | ------------- | -------------------------------- |
| id         | String (cuid) | 主键                             |
| user_id    | String        | 用户 ID                          |
| role       | String        | 角色：`user` / `assistant`       |
| content    | String        | 消息文本内容                     |
| bill_id    | String?       | 关联账单 ID，确认后非空          |
| metadata   | Json?         | 结构化数据：解析结果、确认状态等 |
| created_at | DateTime      | 创建时间                         |

**metadata 结构：**

```typescript
interface AssistantMetadata {
  type: 'confirm_card' | 'guide' | 'confirmed' | 'rejected';
  parseResults?: {
    type: 'expense' | 'income';
    amount: number;
    categoryName: string;
    categoryIcon: string;
    categoryId: string;
    note: string;
    date: string;
    accountName: string;
    accountId: string;
    confidence: 'high' | 'medium' | 'low';
    needsConfirm?: boolean;
    warning?: string;
  }[];
}
```

- `confirm_card`：AI解析成功，展示确认卡片，等待用户确认
- `confirmed`：用户已确认，账单已入库
- `rejected`：用户已取消
- `guide`：解析失败，引导用户

> **多消费设计**: `parseResults` 为数组，支持一次输入多笔消费。单笔消费时数组长度为1，多笔消费时数组长度对应消费笔数。

### 2.4 索引

| 索引     | 字段                  | 用途               |
| -------- | --------------------- | ------------------ |
| 主索引   | user_id + date        | 按用户查询账单列表 |
| 辅助索引 | family_id + date      | 按家庭组查询账单   |
| 辅助索引 | user_id + category_id | 按分类筛选和统计   |
| 辅助索引 | user_id + created_at  | 对话历史按时间排序 |

### 2.5 家庭组关联设计

- 用户创建账单时，自动查询是否在家庭组中
- 在家庭组中 → `family_id` 自动填充为当前家庭组 ID
- 不在家庭组中 → `family_id` 为 null，账单仅属于个人
- 汇总统计时：家庭组模式汇总全家账单，个人模式汇总个人账单

## 三、API 接口

### 3.1 账单接口

#### 创建账单

```
POST /bill
```

| 字段       | 类型   | 必填 | 说明                            |
| ---------- | ------ | ---- | ------------------------------- |
| type       | string | 是   | `expense` 或 `income`           |
| amount     | number | 是   | 金额，最小 0.01                 |
| categoryId | string | 是   | 分类 ID                         |
| note       | string | 否   | 备注                            |
| account    | string | 否   | 账户标签                        |
| date       | string | 否   | 日期，ISO 格式，默认当天        |
| source     | string | 否   | `manual` 或 `ai`，默认 `manual` |

#### 查询账单列表

```
GET /bill?page=1&pageSize=20&type=expense&categoryId=xxx&month=2025-01
```

| 参数                | 类型   | 说明                         |
| ------------------- | ------ | ---------------------------- |
| page                | number | 页码，默认 1                 |
| pageSize            | number | 每页条数，默认 20            |
| type                | string | 筛选类型                     |
| categoryId          | string | 筛选分类 ID                  |
| startDate / endDate | string | 日期范围                     |
| month               | string | 年月筛选，优先级高于日期范围 |

#### 其他账单接口

```
GET    /bill/:id       — 获取详情
PUT    /bill/:id       — 更新账单
DELETE /bill/:id       — 删除账单
GET    /bill/summary   — 月度收支汇总
GET    /bill/today     — 今日收支汇总
```

### 3.2 分类接口

#### 获取分类列表

```
GET /category?type=expense
```

返回系统默认分类 + 用户自定义分类，按系统优先、类型、名称排序。

#### 创建自定义分类

```
POST /category
```

| 字段 | 类型   | 必填 | 说明                  |
| ---- | ------ | ---- | --------------------- |
| name | string | 是   | 分类名称              |
| type | string | 是   | `expense` 或 `income` |
| icon | string | 是   | 图标 emoji            |

#### 更新分类

```
PUT /category/:id
```

仅用户自定义分类可修改，系统默认分类不可修改。

#### 删除分类

```
DELETE /category/:id
```

仅用户自定义分类可删除，有账单引用时拒绝删除。

### 3.3 对话接口

#### 发送消息

```
POST /chat/send
```

| 字段    | 类型   | 必填 | 说明         |
| ------- | ------ | ---- | ------------ |
| content | string | 是   | 用户消息内容 |

**响应：**

```json
{
  "userMessage": { "id": "...", "role": "user", "content": "午饭花了25" },
  "assistantMessage": {
    "id": "msg_xxx",
    "role": "assistant",
    "content": "🍜 餐饮\n支出 ¥25.00",
    "metadata": {
      "type": "confirm_card",
      "parseResults": [{
        "type": "expense",
        "amount": 25,
        "categoryName": "餐饮",
        "categoryIcon": "🍜",
        "categoryId": "cat_xxx",
        "note": "午饭花了25",
        "date": "2025-01-15T12:00:00.000Z",
        "accountName": "",
        "accountId": "",
        "confidence": "high",
        "needsConfirm": true
      }]
    }
  },
  "parseResult": { ... }
}
```

**关键设计：** 解析成功后不立即创建账单，而是将解析结果存入 `metadata`，前端据此渲染确认卡片，用户确认后才创建账单。

#### 获取对话历史

```
GET /chat/history?limit=50&cursor=xxx
```

游标分页，支持翻页加载更早的消息。

#### 确认账单

```
POST /chat/confirm/:messageId
```

| 字段        | 类型   | 必填 | 说明                      |
| ----------- | ------ | ---- | ------------------------- |
| billIndex   | number | 是   | 确认第几笔账单（从0开始） |
| categoryId  | string | 否   | 修改后的分类 ID           |
| amount      | number | 否   | 修改后的金额              |
| note        | string | 否   | 修改后的备注              |
| accountName | string | 否   | 修改后的账户名称          |

用户确认 AI 解析结果中的指定笔账单，从 metadata 取出解析结果，创建账单入库。支持在确认时修改字段。

**响应：**

```json
{
  "confirmed": true,
  "billId": "bill_xxx",
  "bill": { "id": "bill_xxx", "type": "expense", "amount": 25, ... }
}
```

#### 全部确认

```
POST /chat/confirm-all/:messageId
```

用户一次确认 AI 解析结果中的所有账单，支持同时提交编辑内容。

**请求 Body（可选）：**

```json
{
  "edits": {
    "0": { "accountName": "微信" },
    "1": { "amount": 20, "accountName": "支付宝" }
  }
}
```

> `edits` 的 key 为账单索引（从0开始），value 为该笔的修改内容。支持 `categoryId`、`amount`、`note`、`accountName` 字段。

**响应：**

```json
{
  "confirmed": true,
  "billIds": ["bill_xxx", "bill_yyy", "bill_zzz"],
  "bills": [...]
}
```

#### 取消确认卡片

```
POST /chat/reject/:messageId
```

用户取消 AI 解析结果，不创建账单。AI回复取消确认消息。

### 4.4 对话记账核心流程

```
sendMessage():
  1. 保存用户消息
  2. LangGraph 语义解析 → ParseResult[] | null
     失败时降级到 localParse() 关键词解析
  3. 解析成功 → metadata.type = 'confirm_card', metadata.parseResults = [...]
     解析失败 → metadata.type = 'guide'
  4. 保存AI消息（含metadata）
  5. 返回 { userMessage, assistantMessage, parseResults }

confirmBill():
  1. 查找AI消息，取出metadata中的parseResults
  2. 校验metadata.type === 'confirm_card' 和 billIndex 不越界
  3. 合并用户修改（edits）到 parseResults[billIndex]
  4. 设置 parseResults[billIndex].needsConfirm = false
  5. 创建账单（source = 'ai'）
  6. 检查所有 parseResults 是否都已确认
     ├── 全部确认 → 更新 metadata.type = 'confirmed'
     └── 部分确认 → 保持 metadata.type = 'confirm_card'
  7. 更新AI消息metadata
  8. 保存确认对话
  9. 返回 { confirmed: true, billId, bill }

confirmAllBills():
  1. 查找AI消息，取出metadata中的parseResults
  2. 校验metadata.type === 'confirm_card'
  3. 合并用户编辑（edits: Record<number, Partial<ParseResult>>）到对应 parseResults
  4. 处理分类和账户变更的关联更新（categoryName/categoryIcon/accountId）
  5. 过滤出未确认的账单，为每笔创建账单（source = 'ai'）
  6. 更新 metadata → type: 'confirmed'，所有 parseResults.needsConfirm = false
  7. 保存确认对话
  8. 返回 { confirmed: true, billIds, bills }

rejectBill():
  1. 查找AI消息，更新metadata.type = 'rejected'
  2. 保存取消对话
  3. 返回 { rejected: true }
```

### 4.5 本地关键词解析（降级方案）

当 LangGraph LLM 调用失败时，自动降级到本地关键词解析：

1. **多消费拆分**：`splitMultiExpense()` 按逗号/分号/顿号/换行符分割输入，仅当每个分段都包含数字时才拆分为多笔
2. **金额提取**：三种模式
   - 模式1：`数字 + 元/块/rmb/¥`（如"25元"）
   - 模式2：`花了/收入 + 数字`（如"花了25"）
   - 模式3：上下文关键词 + 纯数字（如"午饭25"）
3. **收支判断**：收入关键词（工资/红包/退款等）→ income，否则 → expense
4. **分类匹配**：关键词映射到分类名称，从数据库查询分类ID和图标
   - 午饭/外卖/咖啡 → 餐饮
   - 打车/地铁/滴滴 → 交通
   - 14个分类的关键词映射
5. **日期识别**：支持"昨天"、"前天"
6. **置信度评估**：
   - high：金额明确 + 分类明确
   - medium：金额明确但分类模糊（归为"其他"）
   - low：信息不完整
7. **降级处理**：无法识别时返回引导消息，提示用户使用手动记账

### 4.6 性能优化

- 汇总统计使用 Prisma `aggregate` + `_sum`，数据库层面聚合
- 分类列表查询使用 `OR: [{ isSystem: true }, { userId }]` 一次获取所有分类
- 对话历史使用游标分页，避免深度分页性能问题
- 解析结果存metadata而非额外表，减少数据库查询

### 5.4 确认卡片组件（ConfirmCard）

AI解析成功后，在对话气泡中渲染结构化确认卡片：

**单笔消费模式：**

- **金额行**：分类图标 + 分类名称 + 类型标签 + 金额
- **详情行**：分类选择 + 账户选择 + 日期 + 置信度提示
- **操作行**：取消按钮 + 确认记账按钮
- **已确认状态**：显示"✓ 餐饮 · ¥25.00 · 已记录"

**多笔消费模式：**

- **头部**：识别到N笔消费 + 合计金额
- **每笔卡片**：金额 + 类型标签 + 分类选择 + 账户选择 + 日期 + "确认此笔"按钮
- **底部操作**：取消按钮 + 全部确认按钮
- **单笔确认后**：该笔显示 ✓ 徽章
- **全部确认后**：显示"✓ N笔已记录 · 合计 ¥XX.XX"

### 5.5 手动记账弹窗

点击"手动记账"按钮弹出，分类数据从 CategoryStore 动态加载（系统默认 + 用户自定义），支持支出/收入切换、金额输入、分类选择、账户选择、备注填写。

### 5.6 交互设计

- **对话输入**：底部输入框，支持回车发送，发送后自动滚动到底部
- **确认记账**：确认卡片中的"确认记账"按钮，点击后入账并刷新汇总
- **取消记账**：确认卡片中的"取消"按钮，点击后标记为已取消
- **手动记账**：顶部"手动记账"按钮，弹出完整记账弹窗
- **打字指示器**：AI 处理中显示"思考中..."
- **欢迎消息**：无对话历史时展示引导文案
- **今日收支**：浮动在对话区顶部，记账确认后实时更新

**支出分类（12个）：**

| 分类 | 图标 |
| ---- | ---- |
| 餐饮 | 🍜   |
| 交通 | 🚗   |
| 购物 | 🛒   |
| 娱乐 | 🎮   |
| 居住 | 🏠   |
| 医疗 | 💊   |
| 教育 | 📚   |
