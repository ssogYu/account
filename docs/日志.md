### 2026.5.28

    1. 账户模块优化
        - 移除 Account 模型的 icon 字段（Prisma schema 迁移）
        - 账户管理界面从列表样式改为 tag 样式展示
        - 系统默认账户和用户自定义账户分区展示
        - 清理后端查询和前端组件中的 icon 残留引用

    2. DeepSeek 大模型集成
        - 新增 DeepSeek 配置项（llm.config.ts + .env 环境变量）
        - 使用 @langchain/deepseek 官方包的 ChatDeepSeek 类
        - 解决 DeepSeek V4 思考模式与 tool_choice 不兼容问题（modelKwargs: { thinking: { type: 'disabled' } }）
        - LLM 统一适配层新增 deepseek provider 支持

    3. 多消费解析功能
        - LangGraph BillState 从单消费字段迁移为 bills: BillItem[] 数组
        - LLM Schema 新增 ParseOutputSchema.bills 数组，支持一次返回多笔消费
        - ChatService 新增 confirmAllBills 方法（批量确认所有账单）
        - ChatService.confirmBill 新增 billIndex 参数（支持单笔确认）
        - confirmBill 确认后设置 needsConfirm=false，自动检测是否全部确认
        - localParse 降级方案支持多消费拆分（splitMultiExpense + parseSingleExpense）
        - 移除无用的 ConfirmAllBillsDto，简化 confirm-all 接口

    4. 前端多消费交互
        - ConfirmCard 组件重构：自动识别单笔/多笔模式
        - 多笔消费展示"识别到N笔消费"头部 + 合计金额
        - 支持单笔确认（"确认此笔"按钮）和批量确认（"全部确认"按钮）
        - 单笔确认后显示 ✓ 徽章，全部确认后卡片整体变为已确认状态
        - ChatStore 新增 confirmAllBills action
        - ChatService API 层新增 confirmAllBills 调用

    5. 代码审查修复
        - confirmBill 添加 billIndex 越界校验
        - confirmBill 确认后正确设置 needsConfirm=false
        - allConfirmed 判断从 === false 改为 !（兼容 undefined）
        - 移除 ConfirmCard 中未使用的 anyConfirmed 变量

    6. 多消费确认逻辑优化
        - 恢复高置信度免确认流程：所有账单都满足高置信+有金额+有分类+有账户时自动入库
        - 免确认条件增强：除 confidence/warning 外，额外检查 amount/categoryId/accountName 完整性
        - 允许空金额/空分类进入确认卡片：不过滤 amount=0 或 categoryId='' 的解析结果
        - 金额可编辑：ConfirmCard 金额从 Text 改为 TextInput，支持直接修改
        - 缺失字段高亮：金额为空时橙色提示"请输入金额"，账户未选择时橙色高亮"请选择"
        - 确认前校验：金额必须 > 0 才能确认，否则按钮禁用
        - confirmAllBills 支持 edits 参数：批量确认时可同时提交每笔的编辑内容
        - 新增 ConfirmAllBillsDto 和 BillEditDto
        - ChatStore/ChatService/ChatBubble 全链路适配 edits 参数
