# 统计功能完整实现文档

## 概述

统计页是应用的数据中枢，采用**流水 + 图表**双 Tab 设计。流水视图提供账单管理和筛选，图表视图提供可视化分析。支持图表下钻联动（点击饼图分类自动切换到流水视图并筛选）。所有图表使用纯 React Native View 实现，零 SVG 依赖。

## 3. Server 端实现

### 3.1 API 接口

**文件**: `packages/server/src/modules/bill/bill.controller.ts`

| 接口     | 方法 | 路径                     | 参数              | 说明                     |
| -------- | ---- | ------------------------ | ----------------- | ------------------------ |
| 账单列表 | GET  | `/bill`                  | 分页+筛选参数     | 分页、按月筛选（家庭组） |
| 月度汇总 | GET  | `/bill/summary`          | `month?`          | 月度收支汇总             |
| 分类汇总 | GET  | `/bill/stats/category`   | `month?`, `type?` | 按分类汇总金额和占比     |
| 每日趋势 | GET  | `/bill/stats/daily`      | `month?`          | 按日汇总收支金额         |
| 月度对比 | GET  | `/bill/stats/comparison` | `month?`          | 选中月 vs 上月收支对比   |

### 3.2 核心服务方法

**文件**: `packages/server/src/modules/bill/bill.service.ts`

#### findMany(userId, query) — 账单列表

- 支持分页（page, pageSize）
- 支持按月份筛选（month）
- 支持按类型筛选（type: expense/income）
- 支持按分类筛选（categoryId）
- **使用家庭组筛选**：`buildListWhere` 查询用户所在家庭组的账单
- 返回包含 category 关联的账单列表

#### getCategoryStats(userId, month?, type?)

- 查询指定月份的账单，按分类分组汇总
- 返回每个分类的金额、笔数、占比百分比
- 结果按金额降序排列

#### getDailyStats(userId, month?)

- 查询指定月份每天的收支金额
- 预填充当月所有日期（无数据的天为 0）
- **使用 `toLocalDateKey()` 避免时区偏移**：用 `getFullYear/getMonth/getDate` 代替 `toISOString()`

#### getMonthlyComparison(userId, month?)

- 对比选中月份和上月的收支数据
- 计算支出和收入的环比变化百分比
- **支持月份参数**：根据选中月份计算上月，而非固定当前月

### 3.3 家庭组数据聚合

所有统计方法（包括 `findMany`）统一使用家庭组逻辑：

- 用户已加入家庭组 → 查询 `familyId` 下的所有账单
- 用户未加入家庭组 → 仅查询个人账单

`buildListWhere` 方法已改为 async，内部查询家庭组关系，确保流水列表与统计汇总数据一致。

## 5. 核心交互流程

### 5.1 页面加载

```
1. StatsScreen mount
2. → useEffect: fetchAll()
3. → Promise.all([getSummary, getCategoryStats, getDailyStats])
4. → fetchFlowList() (流水列表单独请求)
5. → fetchMonthlyComparison(month) (传入选中月份)
6. → 数据更新 → 组件渲染
```

### 5.2 月份切换

```
1. 用户点击月份箭头
2. → setSelectedMonth(newMonth)
3. → Store 重置 flowPage/flowGroups
4. → fetchAll(newMonth) 自动触发
5. → 所有数据刷新（包括月度对比）
```

### 5.3 图表下钻联动

```
1. 用户在图表视图点击饼图某个分类
2. → drillToFlow(categoryId, categoryName)
3. → Store 设置 activeTab='flow', drillCategoryId, drillCategoryName
4. → 自动切换到流水视图
5. → fetchFlowList() 带上 categoryId 筛选
6. → 流水视图显示筛选标签 + 该分类的账单
```

### 5.4 左滑操作

```
1. 用户在账单条目上向左滑动
2. → PanResponder 检测水平手势（dx > dy）
3. → translateX 跟随手指移动（0 ~ -140px）
4. → 松手后：超过阈值 → 弹出操作按钮；未超过 → 弹回
5. → 操作按钮露出"编辑"和"删除"
6. → 点击"编辑" → 收回滑动菜单 → 打开 AddBillModal 编辑模式（预填充数据）
7. → 编辑保存 → billService.update() → 刷新所有统计数据
8. → 点击"删除" → Alert 二次确认
9. → deleteBill(billId) → 刷新所有统计数据
```

### 5.5 筛选操作

```
1. 用户点击搜索框展开筛选栏
2. → 选择类型（支出/收入）
3. → setFlowFilter({ type })
4. → Store 重置 flowPage/flowGroups
5. → fetchFlowList() 带上筛选参数
6. → 流水列表按条件过滤
```

## 6. 设计决策

### 6.1 流水/图表双 Tab vs 单页滚动

**选择**: 顶部 Tab 切换

**理由**:

- 流水和图表是两种完全不同的交互模式（管理 vs 分析）
- Tab 切换更清晰，用户心智模型更简单
- 符合项目设计文档 5.3 节的要求

### 6.2 纯 View 图表 vs SVG 库

**选择**: 纯 React Native View 实现

**理由**:

- 避免引入 `react-native-svg` 等重量级依赖（~200KB）
- 堆叠条、进度条、柱状图用 View 即可实现良好视觉效果
- 减少原生链接问题，提升构建稳定性

### 6.3 服务端聚合 vs 客户端聚合

**选择**: 服务端聚合

**理由**:

- 减少网络传输量（只传汇总结果，不传全量账单）
- 利用数据库聚合能力（Prisma aggregate），性能更优
- 客户端只需渲染，逻辑更简单

### 6.4 图表下钻联动

**选择**: 点击饼图分类 → 自动切换流水视图 + 筛选

**理由**:

- 符合项目设计文档 5.3 节要求
- 用户从宏观到微观的自然分析路径
- 下钻标签可一键清除，回到全局视图

### 6.5 PanResponder 左滑 vs 长按

**选择**: PanResponder 手势左滑

**理由**:

- 符合 iOS/Android 原生交互习惯
- 操作路径更短，用户无需长按等待
- 支持跟手拖拽，体验更流畅

## 7. 已修复的问题

| 问题                                                            | 修复方案                                                     |
| --------------------------------------------------------------- | ------------------------------------------------------------ |
| `findMany` 不使用家庭组筛选，流水列表与统计汇总数据不一致       | `buildListWhere` 改为 async，加入家庭组查询逻辑              |

| 账单条目使用长按触发左滑，体验不佳                              | 改用 `PanResponder` 实现跟手左滑手势                         |

| `formatDateLabel` 用 `new Date(dateStr)` 导致 UTC 时区偏移      | 直接解析 YYYY-MM-DD 字符串，用本地中午时间构造 Date          |

| `DailyBarChart` 的 `isToday` 用 `toISOString()` 导致时区偏移    | 改用本地日期方法比较                                         |
| 编辑按钮只收回滑动菜单不跳转                                    | 新增 `onEdit` 回调，打开 `AddBillModal` 编辑模式并预填充数据 |
